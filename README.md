# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 카페 메뉴 주문 및 메뉴 관리 

<br>


## 2. 시나리오 (5~10줄)
* 이 프로그램은 카페에 메뉴를 선택하고 할인 및 결제를 도와주면서 주문 및 결제 분석, 회원 등급 등을 분석하는 시스템입니다.
사용자는 손님이 선택한 메뉴와 단가와 수량을 입력하면 시스템은 이를 바탕으로 총 결제 금액을 산출하고 , 현금 투입 시 거스름돈을 정확하게 계산하여 산출한다. 또한, 누적 결제 금액이나 주문 횟수에 따라 고객등급을 부여하고, 특정 등급 이상을 만족하면 '무료 쿠폰' 발급이나 '포인트 추가 적립' 등의 특별 프로모션이 작동한다. 최종적으로는 사용자는 메인 메뉴판을 통해 주문하기, 매출 통계 보기, 재고 보충하기 등 원하는 기능을 자유롭게 선택하여 카페의 하루 일과를 완벽하게 제어할 수 있다..  

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)

1. 메뉴이름
2. 메뉴가격
3. 주문 수량
4. 단골 할인율
5. 할인 전 총액
6. 할인 적용 후 최종 금액
7. 받은 현금
8. 거스름돈

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
  * 사용자로부터 메뉴 이니셜, 주문 수량, 메뉴 단가, 단골 할인율을 입력받아 총 결제 금액과 거스름돈을 계산하여 출력합니다.
  * 메뉴의 이니셜(문자), 주문수량/단가(정수), 할인율(실수)를 입력받는 인터페이스를 구현했습니다.
  * 사칙연산을 이용해 '할인 적용 후 최종 금액'과 '거스름돈'을 정확하게 계산하여 출력하며, 하단에 손님을 위한 응원 문구를 포함했습니다.
    
### **🤖 AI 파트너십 과정**
1. **내용 1**
    * **프롬프트 요약:** "C언어 4단계 과제 로드맵(입출력 ➔ 조건문 ➔ 반복문 ➔ 배열/구조체)에 적합한 카페 아르바이트 키오스크 시나리오 및 단계별 기능 확장 로드맵 기획 요청"
    * **적용 내용:** 단순한 판매 계산기를 넘어, '단일 주문 및 결제(1차) ➔ 등급별 혜택 및 할인 로직(2차) ➔ 키오스크 메인 메뉴 시스템(3차) ➔ 통합 재고 및 메뉴 관리 솔루션(4차)'으로 이어지는 실무 중심의 시스템 확장 로드맵과 상세 메뉴 구조를 확립함.
 2. **내용 2**
    * **프롬프트 요약:** ""1차 과제 필수 요건(변수 5개 이상, 자료형 3개 이상)을 충족하기 위한 카페 주문 데이터 구조 모델링 논의"
    * **적용 내용:** 단순한 결제 금액 계산을 넘어, 데이터 특성에 맞춰 문자형(char[] - 메뉴 이름/응원 문구), 정수형(int - 단가, 수량, 현금), 실수형(float - 할인율, 부가세)으로 자료형을 명확히 분리함. 이를 통해 실제 키오스크 시스템처럼                      정밀하고 구조적인 입력 데이터 체계를 확립함. )
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** `scanf`로 문자를 입력받을 때 버퍼 문제로 엔터키가 입력되는 현상이 발생함.
     * **해결:** AI에게 질문하여 `scanf(" %c", &initial);` 처럼 `%c` 앞에 공백을 두어 버퍼를 비우는 팁을 얻고 코드를 수정하여 해결함.

## 🟩 [2차 과제: V2.0] 조건 분기 적용 (조건문) - 향후 작성 예정
### **✨2차 과제 업데이트 내용:**
  * 메뉴명 입력시 공백을 허용하도록 설계하였다.
  * 사용자의 **단골 등급**을 입력받아 등급별로 차등화된 할인율(20%, 10%, 5%)을 자동으로 적용하고 if - else if -else 구조를 통해 다양한 사용자 조건에 대응한다.
  * 관계연산자를 이용하여 사용자가 투입한 현금이 최종 결제 금액보다 충분한지 비교하여 결제 승인 여부를 결정하게 하였다
  * 논리연산자를 이용하여 거스름돈이 존재하고 동시에 총 주문 금액이 50,000원 이상인 경우를 판별하여 대량 주문 감사 메시지를 출력하는 특별 로직을 포함했다.
  * 결제 성공 시에만 영수증 출력 및 추가 혜택 검사를 수행하고, 결제 실패 시에는 부족한 금액을 안내하는 중첩구조를 통해 프로그램의 안정성을 강화했다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** 1차 과제에서 했던 카페키오스 코드에 대해서 다중조건문은 단골 할인율 메뉴에 적용한다고 말해주고, 중첩 조건문, 논리/관계연산자를 이용하기 위해서 어떻게 이용하면 좋을지 AI에게 물어보았습니다.
    * **적용 내용:**
    * 다중 조건문을 활용하여 단골등급에 따른 할인율을 적용하였고, 중첩조건문을 통해서는 (1차:메뉴선택 > 2차:재고가 있는가? > 3차:결제 금액이 충분한가?) 메뉴 품절 확인 후 주문 가능 여부 판단을 적용하고, 논리/관계연산자를 이용하여 이벤트 조건을 설정하였습니다.
